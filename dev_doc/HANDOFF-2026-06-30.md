## 1. ✅ 已完成：infer_loop_ensemble.py hang 修了

**不再需要重新查 hang 根因**——上一 agent 已经做完并验证。简要：

- **根因**：`recover_feature_stats` → `extract_features_for_session`，其内层循环每次窗口访问 `csi[f"amplitudes_{name}"].astype(...)`，触发 npz lazy-decompress 3676 次/会话，3 个 session 共 **347 秒**
- **修法**：在 `infer_loop_ensemble.py` 加 `extract_features_fast`（hoist npz 访问到 per-RX dict），改 `recover_feature_stats` 调它
- **验证**：
  - 修复后 2.58 秒（**134× 加速**）
  - X 矩阵 **bit-identical**（`max |X_slow - X_fast| = 0.0`）
  - smoke test：`Models loaded` ~3 秒出，CNN forward 打印 6 类 prob
- **代码变更**：`src/pc_tools/inference/infer_loop_ensemble.py` +85 行，其他文件 0 改

## 2. 你的**第 1 件事**：让 LSTM path 跑起来（**必做，阻塞 ensemble**）

### 2.1 当前阻塞

smoke test 在 20/20 历史 chunks 上打印：
```
[skip chunk_xxx.npz] LSTM feat failed: KeyError
```

**根因**：所有历史 chunks（chunk_20260630_140849_0012 ~ chunk_20260630_141044_0031）都**缺 RX3 数据**——`rx_names=['RX1','RX2','RX3','RX4']`，但 `amplitudes_RX3` / `timestamps_RX3` 不存在。receiver 在 14:08-14:10 那段时间没收到 RX3 板数据（USB 串口断开或板子没插，**未确认**）。

模型是用 4 RX 训的（`CSIClassifier(n_features=16)` = 4 features × 4 RX）。`chunk_to_lstm_features`（`infer_loop_ensemble.py:175-247`）假设 4 RX 都在，访问 `csi[f"timestamps_RX3"]` 抛 KeyError → 整个 chunk LSTM 跳过 → ring buffer 永远凑不齐 16 windows → LSTM 永远 warming-up → ensemble 永远 LSTM-warming-up。

注意：**CNN path 不受影响**——`chunk_to_cnn_spectrogram:137` 已经有 `if f"amplitudes_{name}" not in csi.files: continue` 跳过缺失 RX 的鲁棒性逻辑。

### 2.2 推荐路径（按 ROI）

| 选项 | 描述 | 工时 | 风险 |
|---|---|---|---|
| **A. 跑真 receiver + 30 秒静坐** | 用现有 4 RX 板抓 20+ 个 chunks，确认新数据 4 RX 都齐 → LSTM path 自然 work | 5-10 分钟 | 需物理设备在线 |
| **B. 鲁棒化 `chunk_to_lstm_features`** | 在访问 `timestamps_{name}` / `amplitudes_{name}` 前检查 keys 是否存在；缺失 RX → 填 0 + warn | 15 分钟 | 训练分布与现模型不一致，可能引入 0.05 F1 漂移 |
| **C. 跳过缺 RX 的 chunks** | 在 main loop 加判断：缺任何 RX 就跳过整个 chunk（CNN-only 都不算）| 5 分钟 | 数据利用率低 |

**我的推荐：A → B**。先 A 确认 4 RX 都齐（确认数据通路本身没问题），再 B 加鲁棒性作为长期防御层。

### 2.4 任务 B 的具体做法（如果 A 不达标）

`chunk_to_lstm_features`（`infer_loop_ensemble.py:175`）的两个访问点需要加防御：

```python
# 行 197: t_anchor = csi[f"timestamps_{anchor}"]  → 加 if "timestamps_..." in csi.files
# 行 226: ts = csi[f"timestamps_{name}"] + ...    → 加 if 检查
# 行 227: amps = csi[f"amplitudes_{name}"].astype → 加 if 检查
```

**关键**：模型用 4 RX 训（n_features=16），特征列数和 RX 数耦合。**不能直接 trim 缺 RX 的 chunk**——会破坏列对齐。鲁棒化的做法：

| 方案 | 描述 | 推荐？|
|---|---|---|
| (i) 缺失 RX → 该 RX 的 4 个特征列填 0 + warn | 保持 n_features=16 不变，但训练分布无 0 列 | ⚠ 引入分布漂移 |
| (ii) 缺失 RX → 该 RX 的 4 个特征列填 np.nan + warn | 同上但语义清晰 | ⚠ LSTM 内部 NaN 传播 |
| (iii) 任何 chunk 缺任意 RX → 整个 chunk 跳过（不进 feature_ring）| 干净，但 ring buffer 凑不齐 → LSTM 永远 warming-up | ❌ 实际上行不通 |
| (iv) 加 `--require-rxs` flag：默认 require 4 RX；开 flag 才允许缺 RX | 显式 opt-in，鲁棒性需要时才打开 | ✅ 推荐先 iv，A 跑通再说 |

**我的推荐：先 iv**。A 跑通说明数据本身没问题，B 是"未来防御"——不要现在做。
